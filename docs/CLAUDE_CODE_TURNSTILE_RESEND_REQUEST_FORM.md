# Claude Code Task: Public Request Review Form with Turnstile + Resend

## Goal
Build an automated public request flow for connectNPO.

A nonprofit visitor enters 5 basic fields, passes Cloudflare Turnstile, then the app:
1. creates an organization record,
2. generates/uses the private intake token,
3. sends the private intake link by email using Resend,
4. shows a success screen.

## Project context
- Next.js App Router + TypeScript
- Supabase tables already exist: `organizations`, `intake_responses`, `admin_notes`
- `organizations.intake_token` has a default UUID generated by Supabase
- Existing admin organization creation route: `src/app/admin/organizations/new/*`
- Existing intake route: `src/app/intake/[token]`
- Existing homepage: `src/app/page.tsx`
- Keep sensitive data out of logs and UI.

Add a hidden honeypot field, e.g. `company_website` or `fax_number`. If filled, silently reject or show generic success without sending email.

## Environment variables
Use these env vars:

```text
RESEND_API_KEY
RESEND_FROM_EMAIL
NEXT_PUBLIC_TURNSTILE_SITE_KEY
TURNSTILE_SECRET_KEY
NEXT_PUBLIC_SITE_URL
```

Do not hard-code secrets.

If required env vars are missing, the server action should fail gracefully with a user-friendly message. Do not expose secret names to public users if avoidable.

## Turnstile
- Client renders Turnstile using `NEXT_PUBLIC_TURNSTILE_SITE_KEY`.
- Server verifies token with `TURNSTILE_SECRET_KEY` using:
  `https://challenges.cloudflare.com/turnstile/v0/siteverify`
- Reject failed verification before creating organization or sending email.

## Resend
- Add package if needed: `resend`
- Send email to `contact_email`.
- From: `RESEND_FROM_EMAIL`
- Subject: `Your connectNPO Growth Readiness Intake`
- Email should include a clear button/link to the private intake URL.
- Plain text fallback is okay; simple HTML preferred.

## Database behavior
On valid request:
1. Insert into `organizations` with:
   - name
   - website_url
   - contact_name
   - contact_email
   - contact_role
   - status = `intake_sent`
2. Select `id, intake_token` from inserted row.
3. Build intake link:
   `${NEXT_PUBLIC_SITE_URL}/intake/${intake_token}`
4. Send email through Resend.

If email sending fails after organization insert:
- Update organization status to something like `email_failed` if possible.
- Return a friendly error telling the user to contact connectNPO.
